使用hugo搭建个人博客并托管到github时遇到的问题&总结

October 3, 2021

　　在开始前我对hugo和GitHub一窍不通，只有老师给的一份说明文档。按照说明文档大致上可以完成，但其中遇到的许多问题没有那么容易就找到了解决办法，在我成功搭建博客后的第一篇文章就来记录下我遇到的问题的解决办法以及对于本次搭建博客的总结，以便于以后我继续写博客寻找资料。此文章随时更新。　　

总结

　　首先是对于hugo的解释：　　

　Hugo is a fast and modern static site generator written in Go, and designed to make website creation fun again.　　

　Hugo is a general-purpose website framework. Technically speaking, Hugo is a static site generator. Unlike systems that dynamically build a page with each visitor request, Hugo builds pages when you create or update your content. Since websites are viewed far more often than they are edited, Hugo is designed to provide an optimal viewing experience for your website’s end users and an ideal writing experience for website authors.　　

　　如官网上所说，hugo是一个静态网站生成器。但是每次更新网页必须重新编译整个网站，略有些麻烦，不过目前我也只是在我的博客上记录我所学习的知识笔记等，也不会有多复杂，每次重新编译我觉得没什么大问题。　　

　　那么GitHub又是什么？　　

　GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere.　　

　　Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。GitHub Pages是一个面向开源及私有软件项目的托管平台，其实也就是一个替我管理保存我所有历史代码的平台。因此我用GitHub来托管我的博客就可以实现不断更新博客还能找回历史博客的目的了。　　

1.上传新文章的流程

  • 1)在博客目录下执行 hugo new posts/<post_name>.md 命令，创建一篇的的文章，在content中可以看到。　　

  • 2)执行 hugo server 命令，用来在本地启动一个web服务器，即本地测试效果。这时可以用localhost预览。flag -D表示包括被标记为draft(draft :true)的文章，就是被标记为草稿的文章也会被编译。　　

  • 3)本地测试完成后执行 hugo 命令，该命令会编译生成静态文件并输出到public目录下。　　

  • 4)打开GitHub Desktop根据更改的内容写描述信息并commit，再push到GitHub上。　　

　　

2.本地文件的目录结构

1)content

　　content下的posts文件夹用于存放博客文章。

2)theme

　　该文件夹下存放各种博客主题。

3)config.toml

　　此为配置文件，在这里定义网站主题、网站名称等。

　　

3.MarkDown基本语法

1)分级标题

　　使用[哈希符号#]来表示，哈希符号越少标题字体越大越明显，做多支持六级标题。

2)段落

　　文本内换行最后渲染的效果并没有换行，想要强制换行的话要在上一句末位加上[两个全角的space+enter]。缩进同理，也要用全角的space。

3)分割线

　　使用三个连续的[星号*、减号-或下划线_]来表示分割线。三个符号中间允许有空格，但三个符号渲染后没有区别。

4)列表

　　分为无序列表和有序列表。无序列表在项目前添加[星号*、加号+或减号-]即可。有序列表通过[数字+.]的方式兴义。

5)代码

　　使用[反引号]来定义行内代码。用一对[反引号]包括起代码，如果要在中间插入代码块则直接在首行加一个[制表符tab]或者使用三个连续的[反引号```]作为开始和结束标志。

6)更多语法

　　参考资料:Markdown教程

　　

遇到的问题

1.安装hugo时…

1)系统变量

　　在我已经上传了一篇博客后，再输入命令hugo却显示“hugo不是内部或外部命令”。经检查发现系统变量中path变量中hugo的安装目录不见了，再添加即可。目前不知道是什么原因导致的，但是如果再次出现“hugo不是内部或外部命令”的提示，优先检查path变量。

2)在安装目录下无法使用hugo命令

　　刚开始安装后要退出cmd命令行重新开一个cmd再执行hugo version才能显示安装成功。

　　

2.关于hugo的命令…

1)使用hugo的地方

　　注意要在blog的根目录下使用hugo命令，即connect所在的文件夹，在cmd打开的默认路径下无法使用hugo命令。

2)-D的flag

　　-D的flag表示包括被标记为draft的文章，即也编译标记为draft的文章，被标记为draft的文章也会出现在site中。

　　

3.关于GitHub Desktop…

1)托管路径

　　在GitHub Desktop中打开的是public文件夹，不是connect文件夹。原因看上面总结部分的目录结构。

2)GitHub Desktop commit的时机

　　要在执行hugo命令后才有changes，才能commit。

3)上传失败

　　有时因为网络等问题，github会显示上传失败，个人公开博客上没有最新更改的内容。点击左侧菜单栏的“history”找到要重新提交的更改，右键“Revert…”重新提交即可。
